## 一、项目背景与目标

### 1.1 项目背景

安电通是一个家庭电路服务O2O平台，致力于解决居民家庭电路维修"找不到电工、收费不透明、服务无保障"的痛点。平台连接用户与专业电工，提供应急维修、免费检测、便民安装等服务。

### 1.2 项目目标

| 目标类型 | 指标 | 目标值 |
|---------|------|--------|
| 响应时效 | 首次响应时间 | < 30分钟 |
| 转化效率 | 首访转化率 | ≥ 60% |
| 获客成本 | 线上获客成本(CAC) | ≥ ¥200 |
| 用户价值 | 年均用户价值(LTV) | ≥ ¥1200/年 |
| 服务覆盖 | 目标城市覆盖率 | 首批3个城市 |

### 1.3 产品定位

```
┌─────────────────────────────────────────────────────────┐
│                    安电通产品定位                          │
├─────────────────────────────────────────────────────────┤
│  核心价值: "家庭电路服务的滴滴出行"                         │
│  服务模式: O2O + 平台经济                                │
│  目标市场: 家庭住宅电路服务（非工业/商业）                  │
│  竞争优势: 30分钟响应 + 首次免费检测 + 积分抵物业费         │
└─────────────────────────────────────────────────────────┘
```

#### 4.1.1 功能概述

| 字段 | 内容 |
|------|------|
| 模块名称 | 应急维修 |
| 对应文件 | pages/user/services/Repair.tsx |
| 模块类型 | 业务流程型 |
| Phase-1优先级 | P0 |
| 日均请求量(预估) | 5000+ |

#### 4.1.2 用户流程

```
1. 用户进入应急维修页
2. 选择故障类型（插座冒火/灯不亮/跳闸等）
3. 确认/选择服务地址

5. 提交订单
6. 系统派单/电工抢单
7. 电工接单 → 出发 → 到达 → 施工 → 完工
8. 用户确认并支付
9. 获得积分
```

#### 4.1.3 故障类型配置

| 故障ID | 故障名称 | 图标 | 是否需要上门 |
|--------|---------|------|-------------|
| REPAIR_001 | 插座冒火 | Zap | 是 |

#### 4.1.4 业务规则

| 规则ID | 规则描述 | 优先级 |
|--------|---------|--------|
| RR-001 | 用户下单需选择故障类型 | P0 |
| RR-002 | 用户下单需选择服务地址 | P0 |
| RR-003 | 系统自动派单给3km内在线电工 | P0 |
| RR-004 | 派单超时30秒自动转派 | P1 |
| RR-005 | 电工接单后15分钟未到达可催单 | P1 |
| RR-006 | 完工后用户需确认才能进入支付 | P0 |
| RR-007 | 支付完成后订单结束 | P0 |

#### 4.1.5 状态机

```
PENDING（待接单）
    ↓ 电工接单
ACCEPTED（已接单）
    ↓ 电工点击"到达"
ARRIVED（已到达）
    ↓ 电工点击"开始施工"
IN_PROGRESS（施工中）
    ↓ 电工点击"完工"
COMPLETED（已完成）
    ↓ 用户点击"确认并支付"
PAID（已支付）
    ↓ 系统发放积分
CLOSED（已关闭）

异常路径：
PENDING ──(用户取消)──→ CANCELLED
PENDING ──(超时无人接)──→ NOTIFIED（通知客服）
ACCEPTED ──(电工取消)──→ PENDING（重新派单）
IN_PROGRESS ──(用户投诉)──→ DISPUTE（争议处理）
```

```
│       │                 ↓                     ↓              │
│       │              有问题                无问题            │

│       │         推荐整改方案            检测完成              │

│       │    付费维修                                         │

│       │                                                      │
│       ↓                                                      │

│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### 4.2.3 用户流程

```
1. 用户进入免费检测页
2. 系统判断是否首次检测
   ├── 首次检测 → 显示"首次免费"标识
   └── 非首次 → 显示"¥50上门费"
3. 了解检测内容（漏保测试/插座排查/负载测试）
4. 填写预约信息（姓名/电话/地址/预约时间）
5. 提交预约
6. 系统派单 → 电工接单
7. 电工上门检测
8. 生成检测报告
9. 整改推销（推荐付费维修）
10. 用户选择：仅记录 / 同意整改
11. 如同意整改 → 进入付费维修流程
```

#### 4.2.4 检测项目

| 项目ID | 检测名称 | 检测内容 | 首次免费 | 非首次费用 |

| INS_002 | 全屋开关插座 | 极性检查、松动检查 | ✅ | 含在上门费 |
| INS_003 | 大功率电器 | 负载测试、线路检测 | ✅ | 含在上门费 |
| INS_004 | 线路绝缘性 | 绝缘电阻测试 | ❌ | 额外收费 |
| INS_005 | 配电箱升级 | 更换老旧配电箱 | ❌ | 额外收费 |

#### 4.2.5 首次检测判断规则

| 规则ID | 规则描述 | 优先级 |
|--------|---------|--------|
| IR-000 | **每个用户仅限首次检测免费** | P0 |
| IR-001 | 第二次及后续检测需支付¥50上门费 | P1 |
| IR-002 | 首次检测完成后奖励50积分 | P0 |
| IR-003 | 非首次检测完成后奖励10积分 | P1 |
| IR-004 | 检测完成后必须出具体报告 | P0 |
| IR-005 | 发现问题必须给出整改方案和报价 | P0 |
| IR-006 | 整改转化率纳入电工绩效考核 | P2 |
| IR-007 | 整改订单享受9折优惠 | P1 |

#### 4.2.6 状态机

```
BOOKING（预约中）
    ↓ 预约成功
FOUND（已派单）
    ↓ 电工接单
ACCEPTED（已接单）
    ↓ 电工到达

    ↓
```

#### 4.3.2 用户流程


#### 4.3.3 订单筛选

| 筛选维度 | 选项 | 默认 |
|---------|------|------|

#### 4.3.4 抢单规则

| 规则ID | 规则描述 | 优先级 |
|--------|---------|--------|
| GR-001 | 抢单成功后15秒内不可再抢 | P0 |
| GR-002 | 同一订单只允许抢一次 | P0 |
| GR-003 | 距离优先：同城内距离近优先派单 | P1 |
| GR-004 | 评分优先：高评分电工优先展示 | P1 |
| GR-005 | 电工需开启"听单"状态才能被派单 | P0 |

#### 4.3.5 电工状态

```
OFFLINE（离线）
    ↓ 登录
ONLINE（在线）
    ↓ 开启听单
LISTENING（听单中）
    ↓ 接单/被派单
BUSY（服务中）
    ↓ 服务完成
LISTENING（听单中）
    ↓ 下线
OFFLINE（离线）

异常：
LISTENING ──(长时间无单)──→ 提示提升曝光
BUSY ──(用户投诉)──→ 停单处理
```

| 完成订单 | +订单金额×1 | 1元=1积分 |
| 首次购买 | +50 | 仅首单 |
| 完善资料 | +20 | 仅一次 |
| 签到 | +5 | 每日一次 |
| 好评 | +10 | 每次好评 |

#### 4.4.3 积分消耗规则

| 用途 | 消耗值 | 说明 |
|------|--------|------|
| 抵扣物业费 | 100积分=1元 | 仅限合作物业 |
| 商城购物 | 1积分=1元 | 全品类 |
| 兑换礼品 | 视商品而定 | 积分商城 |

#### 4.4.4 积分计算公式

```
用户获得积分 = 订单实付金额 × 积分倍率

其中：
- 普通订单：积分倍率 = 1.0
- 首次订单：积分倍率 = 1.5（+50%）
- 会员订单：积分倍率 = 1.2~2.0（根据会员等级）
```

| R4 | 首单转化率低 | 中 | 高 | 优化免费检测体验 |
| R5 | 客服能力不足 | 中 | 中 | 接入智能客服 |
| R6 | 竞争对手跟进 | 低 | 中 | 快速建立用户口碑 |
| R7 | 技术选型风险 | 中 | 低 | 使用成熟技术栈 |

### 10.2 详细风险应对

#### R1: 派单逻辑复杂度

```
问题：智能派单涉及距离、评分、技能匹配等多维度计算

应对策略：
- Phase-1：仅实现距离优先派单
- Phase-2：引入评分、好评率等权重
- 预留接口，后续可接入AI派单

触发条件：开发阶段发现复杂度超预期
负责人：技术负责人
```

#### R2: 电工入驻不足

```
问题：平台需要足够电工才能保证服务质量

应对策略：
- 启动期：招募100+电工入驻
- 激励机制：抢单成功奖励、好评奖励
- 收入保障：最低接单收入承诺
- 地推：物业合作、劳务市场推广

触发条件：上线前电工数 < 50
负责人：运营负责人
```

#### R3: 支付资质

```
问题：微信/支付宝支付需要企业资质

应对策略：
- 使用聚合支付（如Ping++）
- 提前2个月申请企业支付资质
- Plan B：上线初期使用充值卡

触发条件：支付资质申请被拒
负责人：商务负责人
```
