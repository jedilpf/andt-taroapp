# 安电通BP框架分析 - 需求确认清单Spec

## Why

用户要求对"汇报材料"、"会议纪要"和"项目逻辑分析"三个文件夹中的所有文件进行系统性深度分析，然后提供一个**简洁清晰的BP（商业计划书）框架**。

## What Changes

### 项目核心信息提取

**项目名称**: 安电通上门电工服务系统
**项目定位**: O2O上门电工服务平台，"家庭电路服务的滴滴"
**核心价值**: 快速匹配 + AI诊断 + 透明定价 + 服务保障

---

### 一、核心业务模块

| 模块 | 说明 | Phase优先级 |
|------|------|-------------|
| 用户端APP | 应急维修、免费检测、便民安装、智慧家居、订单管理、积分商城 | P0 |
| 电工端小程序 | 任务大厅、我的任务、收益管理 | P0 |
| 订单状态机 | 7种状态流转（待接→已接→已到→施工→完成→已支付） | P0 |
| 支付流程 | 微信/支付宝支付 + 积分抵扣 | P0 |
| 积分体系 | 获取（订单/签到/邀请）+ 消耗（抵扣/商城） | P0 |
| 派单系统 | 抢单模式 + 3km优先派单 | P0 |

---

### 二、商业模式

**收入来源**:
- 交易佣金（服务抽成）
- 会员订阅
- 材料代购
- 广告收入
- 企业服务
- 商城收入

**核心竞争优势**:
- 啄木鸟抽成40-50%，安电通前期约20-25%（减半吸引电工）
- 首次上门免费检测 → 后续一次性收费（不重复收费）
- AI辅助诊断，提升用户信任

**用户粘性策略**:
- 积分系统：积分可抵物业费（核心留存点）
- 商城消费闭环
- 签到激励 + 连续打卡奖励

---

### 三、市场规划

**目标用户**:
- 租房年轻用户（通过房东间接服务）
- 35-55岁上班族家庭用户
- 55岁以上银发用户（核心用户）

**获客渠道**:
- 社区街道和物业（最核心，获客成本最低）
- 应用商店自然量
- 微信生态
- 口碑传播

**市场数据**:
- 家政服务市场持续扩大
- 电工服务占比小但潜力大
- 3.3亿60岁以上老年人存在用电安全需求

---

### 四、开发规划

**技术栈**:
- 前端：React + TypeScript + Taro（多端复用）
- 后端：微服务架构（用户/订单/支付/地图/消息）
- AI：Gemini/DeepSeek辅助诊断

**开发周期（满员8人团队）**:
- 阶段一（2-3周）：技术选型 + 架构设计
- 阶段二（6-8周）：后端核心模块开发
- 阶段三（4-6周）：前后端联调
- 阶段四（3-4周）：测试与验收
- 阶段五（2-3周）：上线准备
- **总工期：约17-24周（4-6个月）**

**Q1-Q4规划**:
- Q1：Demo验证 + 技术选型
- Q2：后端开发 + 联调
- Q3：内测 + 小范围试运营
- Q4：V1.0正式上线

---

### 六、BP框架结构

根据用户需求（项目规划/开发周期/项目成本/市场规划/营收模式），建议BP框架如下：

```
1. 项目概述
   - 项目名称与定位
   - 核心价值主张
   - APP内容介绍

2. 市场分析
   - 行业背景与痛点
   - 目标用户群体
   - 市场规模预估
   - 竞品分析

3. 产品规划
   - 核心功能介绍
   - 开发周期与里程碑
   - 技术架构

4. 商业模式
   - 营收模式（第一阶段商业化价值）
   - 定价策略
   - 用户粘性策略

5. 运营策略
   - 市场规划（前期推广占比）
   - 获客渠道
   - 社区/物业合作模式

6. 财务规划
   - 项目成本（开发费用 + 预算）
   - 收入预估
   - LTV模型

7. 团队介绍
   - 核心成员
   - 组织架构

8. 发展愿景
   - 短期目标
   - 长期愿景
```
