## 四、核心技能设计

### 4.1 需求分析技能 (requirement-analyzer)

```yaml
---
name: requirement-analyzer
description: 需求分析智能体，自动解析需求并生成任务分解
version: 1.0.0
metadata:
  openclaw:
    requires:
      env:
        - GITHUB_TOKEN
        - DATABASE_URL
      bins:
        - git
    primaryEnv: GITHUB_TOKEN
---

# 需求分析智能体

## 角色定义
你是一个专业的需求分析师，帮助团队将模糊的需求转化为清晰的任务列表。

## 输入格式
用户需求描述，例如：
- "添加用户登录功能"
- "接入微信支付"
- "优化首页加载速度"

## 输出格式
1. **需求理解**：复述并澄清需求
2. **任务分解**：
   | 任务ID | 任务名称 | 负责人 | 工时估算 | 依赖 |
   |-------|---------|-------|---------|-----|
   | T001  | xxx     | 前端  | 2人天   | -   |
3. **风险点**：潜在的技术风险或业务风险
4. **建议优先级**：P0/P1/P2

## 工具调用
- 调用GitHub API创建Issue
- 调用数据库查询现有任务
- 生成Markdown文档
```

### 4.2 进度跟踪技能 (progress-tracker)

```yaml
---
name: progress-tracker
description: 进度跟踪智能体，监控项目进度并生成报告
version: 1.0.0
metadata:
  openclaw:
    requires:
      env:
        - DATABASE_URL
        - SLACK_WEBHOOK
    primaryEnv: DATABASE_URL
---

# 进度跟踪智能体

## 角色定义
你是一个项目经理，负责监控项目进度，识别延迟风险。

## 监控维度
1. **任务完成率**：已完成/总数
2. **里程碑进度**：当前里程碑完成情况
3. **资源利用率**：人员工作量分布
4. **延期风险**：可能延期的任务列表

## 输出格式
```
📊 项目进度报告 - 2026-04-08

整体进度：████████░░ 80%

里程碑状态：
- M1 框架搭建 ✅ 已完成
- M2 API开发 🔄 进行中（延迟1天）
- M3 小程序开发 ⏳ 待开始

风险预警：
⚠️ T005 支付API 延迟风险高
   原因：微信支付文档更新
   建议：安排后端专项攻坚

团队工作负载：
- 前端：████████░ 80%
- 后端：██████████ 100%（满载）
- 运维：████░░░░░░ 40%
```

## 触发条件
- 每日定时（早9点）
- 关键任务状态变更
- 手动触发 "/进度"
```

### 4.3 文档生成技能 (doc-generator)

```yaml
---
name: doc-generator
description: 文档自动生成智能体，根据代码和任务生成技术文档
version: 1.0.0
metadata:
  openclaw:
    requires:
      bins:
        - git
        - node
---

# 文档生成智能体

## 能力范围
1. **API文档生成**：从Controller代码生成API文档
2. **数据库文档更新**：从DDL语句生成表结构文档
3. **更新日志生成**：从Git提交记录生成CHANGELOG
4. **部署文档生成**：从Docker配置生成部署指南

## 使用方式
- "/doc api" - 生成API文档
- "/doc db" - 更新数据库文档
- "/doc changelog" - 生成更新日志
```

## 五、安装部署方案

### 5.1 环境准备

```bash
# 安装Node.js 24
nvm install 24
nvm use 24

# 安装OpenClaw
npm install -g openclaw@latest

# 初始化配置
openclaw onboard --install-daemon

# 配置模型（使用OpenAI或国内API）
openclaw config set model "gpt-4o"
```

### 5.2 安装项目管理技能

```bash
# 克隆技能仓库
git clone https://github.com/andiantong/pm-skills.git

# 发布技能到本地
cd pm-skills
openclaw skill publish requirement-analyzer
openclaw skill publish progress-tracker
openclaw skill publish doc-generator

# 同步技能
openclaw sync
```

### 5.3 配置通知渠道

```bash
# 配置微信通知（企业微信机器人）
openclaw channel add wechat --webhook "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx"

# 配置钉钉通知
openclaw channel add dingtalk --webhook "https://oapi.dingtalk.com/robot/send?access_token=xxx"
```

## 六、与安电通项目集成

### 6.1 集成点

| 集成项 | 方式 | 数据流向 |
|-------|------|---------|
| GitHub Issues | API | 双向同步 |
| 项目数据库 | MySQL | 读取任务状态 |
| 文档仓库 | Git | 自动更新 |
| 通知渠道 | Webhook | 发送预警 |

### 6.2 使用场景

```
【场景1】新需求分析
用户 @PM-Bot：添加广告模块
    ↓
OpenClaw → 需求分析Skill
    ↓
输出：
├── T011 广告位设计（0.5人天）
├── T012 广告表设计（0.5人天）
├── T013 广告API开发（2人天）
├── T014 前端广告组件（1.5人天）
└── 风险：广告合规审核
    ↓
自动创建GitHub Issues

【场景2】每日进度报告
每日9:00 → OpenClaw定时触发
    ↓
进度跟踪Skill
    ↓
查询数据库 → 分析进度 → 生成报告
    ↓
发送到微信群、钉钉群

【场景3】风险预警
检测到T005延迟 → 自动预警
    ↓
@项目负责人 T005支付API延迟风险
原因：微信支付文档更新
建议：安排后端专项攻坚
```
