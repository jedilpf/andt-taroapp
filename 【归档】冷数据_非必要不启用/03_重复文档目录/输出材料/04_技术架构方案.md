### 1.2 架构设计原则

| 原则 | 说明 | 实现方式 |
|-----|------|---------|
| 简单优先 | MVP阶段避免过度设计 | 单体架构、模块化设计 |
| 快速迭代 | 降低开发复杂度 | 统一代码库、简化部署 |
| 可扩展 | 为未来演进预留空间 | 模块化拆分、接口抽象 |
| 高性能 | 响应时间<500ms | Redis缓存、异步处理 |
| 安全性 | 数据安全、接口安全 | JWT认证、参数校验 |
| 可维护 | 代码清晰、文档完善 | 统一规范、注释文档 |

### 1.3 架构选型理由

| 对比项 | 单体架构（当前选择） | 微服务架构 |
|-------|-------------------|-----------|
| 开发效率 | 高，统一代码库 | 低，多服务协调 |
| 部署复杂度 | 低，单应用部署 | 高，多服务编排 |
| 运维成本 | 低，单服务器即可 | 高，需多节点部署 |
| 团队要求 | 1-2人即可维护 | 需专职运维 |
| 扩展能力 | 垂直扩展为主 | 水平扩展灵活 |
| 适用阶段 | MVP/初创期 | 成熟期/高并发 |

**选择依据**：第1期（M1-M6）用户规模<500人，单服务器足够支撑，单体架构可降低70%开发和运维成本。

### 3.4 API请求封装

```typescript
// request.ts - HTTP请求封装
import axios from 'axios';

const request = axios.create({
  baseURL: process.env.API_BASE_URL,
  timeout: 10000,
});

// 请求拦截器 - 添加Token
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器 - 统一错误处理
request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      // Token过期，跳转登录
    }
    return Promise.reject(error);
  }
);
```

### 4.3 统一响应格式

```java
// Result.java - 统一响应封装
public class Result<T> {
    private Integer code;       // 状态码
    private String message;     // 提示信息
    private T data;             // 业务数据
    private Long timestamp;     // 时间戳

    public static <T> Result<T> success(T data) {
        return new Result<>(200, "success", data, System.currentTimeMillis());
    }

    public static <T> Result<T> error(Integer code, String message) {
        return new Result<>(code, message, null, System.currentTimeMillis());
    }
}
```

### 4.4 模块间调用

```java
// 模块间调用 - 直接注入Service
@Service
public class OrderServiceImpl implements OrderService {
    
    @Autowired
    private UserService userService;  // 直接注入用户服务
    
    @Autowired
    private AddressService addressService;  // 直接注入地址服务

    public OrderVO getOrderDetail(Long orderId) {
        Order order = orderMapper.selectById(orderId);
        User user = userService.getById(order.getUserId());
        Address address = addressService.getById(order.getAddressId());
        // 组装返回
        return OrderVO.build(order, user, address);
    }
}
```

```
┌──────────────────────────────────────────────────┐
```

### 7.2 JWT配置


### 7.3 安全措施清单

### 8.2 支付回调处理

```java
// 支付回调处理流程
@Service
public class PaymentCallbackService {

    @Transactional
    public void handleCallback(PaymentCallbackDTO callback) {
        // 1. 验证签名
        if (!verifySignature(callback)) {
            throw new SecurityException("签名验证失败");
        }

        // 2. 查询支付记录
        Payment payment = paymentMapper.selectByOrderNo(callback.getOrderNo());

        // 3. 防重复处理
        if (payment.getStatus() == PAID) {
            return; // 已处理
        }

        // 4. 更新支付状态
        payment.setStatus(PAID);
        paymentMapper.updateById(payment);

        // 5. 更新订单状态
        orderService.updateStatus(payment.getOrderId(), PAID);

        // 6. 发送通知（同步处理，MVP阶段简化）
        messageService.sendPaymentSuccessNotice(payment.getUserId());
    }
}
```
