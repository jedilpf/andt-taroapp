一、OpenClaw 核心设计思路解析

1.1 架构哲学

设计原则OpenClaw实现核心思想
本地优先Gateway在本地运行数据和计算在用户设备，不依赖云端
控制平面分离Gateway是控制层，Assistant是产品基础设施服务于用户体验
多通道统一22+消息渠道统一接入一个Assistant覆盖所有场景
职责分离Gateway协调，Node执行关注点分离，便于扩展
安全默认权限门控、DM pairing安全不是可选项

// PM-Gateway 核心职责 interface PMGateway { // Session管理（借鉴OpenClaw Session模型） sessions: { main: Session; // 主会话 projects: Map<string, Session>; // 项目隔离会话 }; // Channel路由（借鉴OpenClaw多通道） channels: { wechat: ChannelAdapter; dingtalk: ChannelAdapter; feishu: ChannelAdapter; cli: ChannelAdapter; web: ChannelAdapter; }; // Tool调度（借鉴OpenClaw Tool系统） tools: { invoke(skill: string, input: any): Promise<any>; list(): Skill[]; publish(skill: Skill): void; }; // 状态持久化 persistence: { sessionConfig: SessionConfig; context: ProjectContext; history: Message[]; }; } // Session模型（借鉴OpenClaw） interface Session { id: string; type: 'main' | 'project'; config: { thinkingLevel: 'low' | 'medium' | 'high'; verboseLevel: 'quiet' | 'normal' | 'verbose'; model: string; }; context: ProjectContext; queue: MessageQueue; activation: 'auto' | 'manual'; }

3.2 PM-Skills 三层架构

# bundled skill: requirement-analyzer name: requirement-analyzer description: 需求分析智能体 version: 1.0.0 type: bundled # 内置技能 config: requires: env: - OPENAI_API_KEY # 或国内API tools: - github-api - database-query permissions: - read:tasks - write:tasks - create:issues behavior: input: type: object properties: requirement: { type: string } context: { type: object } output: type: object properties: understanding: { type: string } tasks: { type: array } risks: { type: array } priority: { type: string, enum: [P0, P1, P2] } steps: - name: understand action: llm.invoke - name: decompose action: llm.invoke - name: validate action: approval.gate message: "确认任务分解方案？"

3.3 PM-Workflows 工作流引擎

# 工作流：新功能开发全流程 name: feature-development-pipeline description: 新功能开发全流程自动化 args: requirement: { type: string, required: true } deadline: { type: date, required: false } steps: # Step 1: 需求分析 - id: analyze skill: requirement-analyzer input: { requirement: $args.requirement } output: [tasks: array, risks: array, estimate: object] # Step 2: 人工审批门控 - id: approve_tasks approval: message: | 需求分析完成，任务分解如下： {{ $analyze.tasks | table }} 确认执行？ channels: [wechat, dingtalk] when: $analyze.tasks # Step 3: 创建任务 - id: create_tasks skill: task-creator input: $analyze.tasks when: $approve_tasks.approved # Step 4: 排期建议 - id: schedule skill: auto-scheduler input: { tasks: $create_tasks.ids, deadline: $args.deadline } when: $create_tasks.success # Step 5: 通知团队 - id: notify skill: team-notifier input: message: | 新功能开发任务已创建： 需求：{{ $args.requirement }} 任务数：{{ $create_tasks.count }} 预计完成：{{ $schedule.endDate }} channels: [wechat, dingtalk]

3.4 PM-Nodes 设备节点

// Git Node - 本地Git操作 class GitNode implements PMNode { capabilities = { name: 'git-node', actions: [ { name: 'status', description: '查看Git状态' }, { name: 'commit', description: '提交代码', requiresApproval: true }, { name: 'push', description: '推送代码', requiresApproval: true }, { name: 'create-branch', description: '创建分支' }, { name: 'create-pr', description: '创建PR', requiresApproval: true } ], permissions: ['git:read', 'git:write'] }; async invoke(action: string, params: any) { switch (action) { case 'status': return exec('git status'); case 'commit': if (params.requireApproval) { await this.requestApproval(`确认提交: ${params.message}`); } return exec(`git commit -m "${params.message}"`); } } } // Notify Node - 通知发送 class NotifyNode implements PMNode { capabilities = { name: 'notify-node', actions: [ { name: 'send-wechat', description: '发送微信通知' }, { name: 'send-dingtalk', description: '发送钉钉通知' }, { name: 'send-email', description: '发送邮件' } ], permissions: ['notify:send'] }; }

7.1 为什么借鉴OpenClaw

理由说明
成熟架构35万+Stars验证
本地优先数据安全，响应快

差异点原因
