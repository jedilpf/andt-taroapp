安电通API接口文档

未来申活（上海）数字科技有限公司

编制日期：2026年4月8日

架构说明：本项目采用单体架构设计，所有API接口由同一后端应用统一提供。接口按业务模块进行分组管理，便于开发和维护。

一、接口规范

1.1 基础信息

项目说明
基础URLhttps://api.andiantong.com
协议HTTPS
数据格式JSON
字符编码UTF-8
时间格式yyyy-MM-dd HH:mm:ss

1.2 请求规范

请求头: Content-Type: application/json Authorization: Bearer {token} // 需要认证的接口 X-Request-Id: {uuid} // 请求追踪ID（可选）

1.3 响应格式

成功响应: { "code": 200, "message": "success", "data": { ... }, "timestamp": 1712505600000 } 错误响应: { "code": 400, "message": "参数错误", "data": null, "timestamp": 1712505600000 }

1.4 状态码定义

状态码说明
200成功
400参数错误
401未认证/Token过期
403无权限
404资源不存在
500服务器错误

二、用户模块

POST /api/user/register 用户注册

参数类型必填说明
phoneString是手机号(11位)
passwordString是密码(6-20位)
codeString是验证码(6位)
响应: { userId, phone, token }

POST /api/user/login 用户登录

参数类型必填说明
phoneString是手机号
passwordString是密码
响应: { userId, nickname, avatar, phone, token, refreshToken }

POST /api/user/wechat-login 微信登录

参数类型必填说明
codeString是微信授权code

GET /api/user/info 获取用户信息

请求头: Authorization: Bearer {token} 响应: { userId, nickname, avatar, phone, gender, role, createdAt }

PUT /api/user/update 更新用户信息

参数类型必填说明
nicknameString否昵称
avatarString否头像URL
genderInteger否性别(1男/2女)

POST /api/user/validate-token Token验证

参数类型必填说明
tokenString是JWT Token
响应: { valid, userId, expireAt }

POST /api/user/send-code 发送验证码

参数类型必填说明
phoneString是手机号
typeInteger是类型(1注册/2登录/3重置密码)

POST /api/user/reset-password 重置密码

参数类型必填说明
phoneString是手机号
codeString是验证码
newPasswordString是新密码

三、订单模块

POST /api/order/create 创建订单

参数类型必填说明
serviceTypeInteger是服务类型(1维修/2安装/3安检)
serviceNameString是服务名称
descriptionString否问题描述
addressIdLong是地址ID
contactPhoneString是联系电话
appointmentTimeString否预约时间
amountDecimal是订单金额
响应: { orderId, orderNo, status, createdAt }

GET /api/order/list 订单列表

参数类型必填说明
statusInteger否订单状态
pageNumInteger否页码(默认1)
pageSizeInteger否每页数量(默认10)
响应: { total, list: [{ orderId, orderNo, serviceName, amount, status, statusText, elecName, appointmentTime, createdAt }] }

GET /api/order/{id} 订单详情

响应包含: 订单信息、用户信息、电工信息、地址信息、评价信息

POST /api/order/{id}/cancel 取消订单

参数类型必填说明
reasonString否取消原因
响应: { orderId, status, refundAmount }

POST /api/order/{id}/status 订单状态变更

参数类型必填说明
statusInteger是目标状态
remarkString否备注

POST /api/order/{id}/review 订单评价

参数类型必填说明
ratingInteger是评分(1-5)
contentString否评价内容
tagsArray否评价标签

GET /api/order/{id}/logs 订单日志

响应: [{ action, fromStatus, toStatus, operatorType, createdAt }]}

订单状态枚举

状态值状态名说明
-1已取消订单已取消
0待支付订单已创建，等待支付
10已支付支付成功，等待电工接单
20已接单电工已接单，前往服务
30服务中电工正在服务
40已完成服务完成，等待评价
50已评价用户已评价，订单结束

十一、广告模块

说明：广告接口不需要登录，但需要deviceId用于统计。点击上报返回跳转链接，由前端处理跳转逻辑。

GET /api/ad/splash 获取开屏广告

参数类型必填说明
deviceIdString是设备唯一标识
响应: { id, imageUrl, videoUrl, linkUrl, linkType, duration, skipAfter } 字段说明: - id: 广告ID - imageUrl/videoUrl: 图片或视频广告URL - linkUrl: 点击跳转链接 - linkType: 跳转类型(1页面跳转/2外链/3商品详情/4服务预约) - duration: 广告展示时长(秒) - skipAfter: 可跳过时间(秒)

GET /api/ad/banner 获取横幅广告列表

参数类型必填说明
positionString是广告位编码(如home_banner)
响应: [{ id, title, imageUrl, linkUrl, linkType }] 常见广告位: home_banner(首页), store_banner(商城), service_banner(服务页)

POST /api/ad/{id}/show 广告展示上报

参数类型必填说明
deviceIdString是设备唯一标识
路径参数: id(广告ID) 响应: { success: true }

POST /api/ad/{id}/click 广告点击上报

参数类型必填说明
deviceIdString是设备唯一标识
路径参数: id(广告ID) 响应: { linkUrl, linkType } - 返回跳转链接，由前端处理跳转

跳转类型枚举(linkType)

值说明前端处理
1页面跳转跳转App内指定页面
2外链跳转打开外部浏览器
3商品详情跳转商品详情页
4服务预约跳转服务预约页

十三、接口调用流程

完整下单流程

1. 用户登录 → POST /api/user/login → 获取token 2. 获取地址 → GET /api/address/list → 选择地址 3. 创建订单 → POST /api/order/create → 获取订单ID 4. 发起支付 → POST /api/payment/create → 获取支付参数 5. 支付回调 → 系统自动处理 6. 电工接单 → POST /api/electrician/accept/{orderId} → status=20 7. 开始服务 → POST /api/order/{id}/status → status=30 8. 完成服务 → POST /api/order/{id}/status → status=40 9. 用户评价 → POST /api/order/{id}/review → 订单结束

电工端接单流程

1. 电工登录 → POST /api/user/login → 获取token 2. 更新位置 → POST /api/map/location/update → 上报当前位置 3. 开启接单 → PUT /api/electrician/status → status=1 4. 浏览任务大厅 → GET /api/electrician/hall → 获取附近订单 5. 接单 → POST /api/electrician/accept/{orderId} → 订单分配 6. 前往服务 → POST /api/order/{id}/status → status=30 7. 完成服务 → POST /api/order/{id}/status → status=40 8. 查看收入 → GET /api/electrician/info → 查看累计收入
